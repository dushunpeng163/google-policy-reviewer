# Web 界面使用指南

[English Guide](WEB_GUIDE_EN.md)

## 启动

```bash
python3 launcher.py --mode web
# 浏览器打开 http://localhost:8080
```

---

## 主界面

打开后看到两个入口卡片：

**「开发新游戏」**
→ 填写游戏名称、类型、目标用户年龄、功能、市场、平台  
→ 点击「生成合规指南」  
→ 查看分阶段开发路线图和平台配置清单

**「检测已有游戏」**
→ 填写游戏信息，点击「📁 浏览」选择 Unity 项目路径  
→ 点击「开始检测」  
→ 查看代码扫描结果和合规问题列表（含文件路径和行号）

---

## 政策管理弹窗

点击右上角「📋 政策管理」按钮打开，共四个 Tab：

---

### 📊 状态 Tab

显示 22 条核心政策规则的新鲜度：

- **整体状态**：全部新鲜 / 待复核 / 严重过期
- **LLM 变化告警**：当检测到政策实质变化时，告警卡片自动出现，包含：
  - LLM 生成的变化摘要
  - 对开发者的具体影响描述
  - 建议操作
  - 检测时间和来源
- **标记已复核**：每个告警卡片有两个按钮：
  - ✅「已评估，规则无需修改」：确认告警，清除待复核状态
  - ✏️「已修改规则，清除告警」：确认已根据政策变化更新了规则
- **全部规则详情**：可展开查看每条规则的状态、年龄和官方文档链接

---

### ⚙️ 配置 Tab

**LLM API Key**

| 字段 | 说明 |
|------|------|
| Anthropic API Key | Claude（推荐，用于政策分析） |
| OpenAI API Key | GPT-4o（备选） |

填入后点击「保存配置」即时生效，无需重启服务器。  
清空字段后保存可删除 Key。

**合规规则热更新**

| 字段 | 说明 |
|------|------|
| 远程规则 URL | 可选，指向远端 `compliance-rules.yaml` 的 URL |
| 当前版本 | 当前加载的规则文件哈希 |

- 不填 URL：本地模式，系统监控 `config/compliance-rules.yaml` 文件变化自动重载
- 填写 URL：远程模式，系统自动拉取并应用更新
- 点击「检查更新」：立即触发一次更新检查

---

### 🔍 检查 Tab

三种检查方式，均在**后台运行**（切换 Tab 不会中断检查）：

| 按钮 | 动作 |
|------|------|
| 📡 RSS 公告 | 抓取 Apple/Google 开发者博客，过滤政策相关条目 |
| 🔍 页面变化检测 | 对比官方政策页面内容哈希，检测变化 |
| ⚡ 全部检查 | RSS + 页面检测 + LLM 自动分析（一键完成） |

**执行过程：**

1. 点击按钮后任务立即在后台启动
2. 界面每 2 秒自动轮询进度（"已等待 N 秒"）
3. 完成后自动展示结果，即使中途切换过 Tab

**已配置 LLM 时**，全部检查额外完成：
- 分析页面变化的具体内容
- 识别受影响的规则
- 在状态 Tab 中自动将受影响规则标记为「待复核」

**未配置 LLM 时**，仍可检测页面哈希变化，但无法自动识别受影响规则。

---

### ⏰ 定时 Tab

配置自动后台监控：

**调度器状态**：运行中 / 已停止，显示当前检查间隔

**可选频率**：
- 每 6 小时
- 每 12 小时
- 每 24 小时（推荐）
- 每周一次

**通知历史**：列出最近的政策变化告警和时间戳。点击「全部标为已读」清除未读 badge。

定时器按设定频率自动执行全部检查逻辑。结果通过顶部通知铃铛（🔔）推送，badge 显示未读数量。

---

## 底部动作按钮

弹窗底部右侧主按钮随当前 Tab 动态变化：

| 当前 Tab | 按钮 |
|---------|------|
| 📊 状态 | 🔄 刷新状态 |
| ⚙️ 配置 | 💾 保存配置 |
| 🔍 检查 | ⚡ 一键全部检查 |
| ⏰ 定时 | ✅ 全部标为已读 |

---

## 通知铃铛（🔔）

- 位于顶部右侧，badge 显示未读通知数量
- 触发通知的场景：
  - 定时检查发现 RSS 政策公告
  - 检测到页面哈希变化
  - LLM 将规则标记为待复核
- 在「定时」Tab 中查看和清除通知
