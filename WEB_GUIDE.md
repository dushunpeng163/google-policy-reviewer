# Web 界面使用指南

## 启动

```bash
python3 launcher.py --mode web
# 浏览器打开 http://localhost:8080
```

---

## 主界面

打开后看到两个入口卡片：

**「开发新游戏」**
→ 填写游戏名称、类型、目标用户年龄、功能、市场、平台
→ 点击「生成合规指南」
→ 查看分阶段开发路线图和平台配置清单

**「检测已有游戏」**
→ 填写游戏信息，点击「📁 浏览」选择 Unity 项目路径
→ 点击「开始检测」
→ 查看代码扫描结果和合规问题列表

---

## 政策管理弹窗

点击右上角「📋 政策管理」按钮打开，共三个 Tab：

### 📊 状态 Tab
- 显示 22 条核心政策规则的新鲜度
- 整体状态（全部新鲜 / 待复核 / 严重过期）
- 可展开查看每条规则详情和官方文档链接

### ⚙️ 配置 Tab
- 输入 Anthropic Claude 或 OpenAI GPT-4o 的 API Key
- 点击「💾 保存配置」即时生效，**无需重启服务器**
- 已配置时显示 Key 前缀，可随时更新或清除

### 🔍 检查 Tab
三种检查方式：

| 按钮 | 动作 |
|------|------|
| 📡 RSS 公告 | 抓取 Apple / Google 开发者博客，过滤政策相关条目 |
| 🔍 页面变化检测 | 对比官方政策页面哈希，检测内容变化 |
| ⚡ 全部检查 | RSS + 页面检测 + LLM 自动分析（一键完成） |

检测到变化且已配置 LLM 时，自动显示分析结果：
- 变化摘要
- 影响等级（critical / high / medium / low）
- 对你游戏的具体影响
- 建议操作

---

## 底部按钮说明

弹窗底部右侧按钮随当前 Tab 动态变化：

- 📊 状态 → **「🔄 刷新状态」**（重新拉取新鲜度数据）
- ⚙️ 配置 → **「💾 保存配置」**（直接触发保存）
- 🔍 检查 → **「⚡ 一键全部检查」**（触发完整检查流程）
