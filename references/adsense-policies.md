# Google AdSense 广告政策详解

## 核心政策要求

### 📊 广告展示标准

#### 广告位置
- **允许**: 内容周围、文章间、页面顶部/底部
- **限制**: 不得遮挡主要内容
- **禁止**: 浮动广告覆盖操作按钮

#### 广告密度
- 每屏最多3个广告单元
- 广告内容比例不得超过50%
- **移动端特殊要求**: 首屏不超过1个广告

#### 用户体验
- 广告必须有明确的"广告"标识
- 不得诱导点击（"点击这里"等）
- 关闭按钮必须清晰可见

### 🚫 禁止内容

#### 成人内容
- 色情、裸体图像
- 性暗示内容
- 约会应用中的露骨内容

#### 暴力内容
- 血腥、残忍画面
- 恐怖、惊悚内容
- 真实暴力事件描述

#### 非法内容
- 盗版软件、破解工具
- 毒品相关内容
- 赌博和博彩

### 💰 点击欺诈预防

#### 禁止行为
- 自己点击广告
- 鼓励他人点击
- 使用机器人刷点击

#### 检测机制
- Google AI自动检测异常流量
- IP地址和设备指纹分析
- 点击模式异常识别

## 移动应用特殊要求

### 📱 应用内广告规范

#### Banner广告
```
位置：屏幕顶部或底部
大小：320x50, 320x100, 728x90
要求：不覆盖操作区域
```

#### 插屏广告
```
触发：自然暂停点（关卡间、页面切换）
频率：不超过每分钟1次
关闭：必须提供关闭按钮
```

#### 原生广告
```
标识：必须标注"广告"或"Sponsored"
样式：与应用内容风格一致
位置：内容流中的自然位置
```

#### 激励视频
```
长度：15-30秒
奖励：观看完成后给予
选择：用户可选择不观看
```

### 🔧 技术实现要求

#### AdMob SDK集成
- 使用最新版本SDK
- 正确配置广告单元ID
- 实现错误处理机制

#### 广告加载
- 预加载机制优化体验
- 网络错误时的降级处理
- 广告请求频率控制

## 违规处理流程

### ⚠️ 警告阶段
- 邮件通知违规内容
- 30天整改期限
- 继续获得广告收入

### 🚨 限制阶段
- 部分广告功能受限
- 收入可能受影响
- 需要提交申诉

### ❌ 封号阶段
- 完全停止广告服务
- 未支付收入可能没收
- 申诉成功率较低

## 最佳实践建议

### ✅ 推荐做法
1. **合理广告密度**: 保持内容和广告平衡
2. **用户友好体验**: 不影响核心功能使用
3. **高质量内容**: 原创、有价值的应用内容
4. **定期政策审查**: 跟进政策更新

### 💡 收入优化技巧
1. **A/B测试**: 测试不同广告位置和大小
2. **用户分析**: 了解用户行为优化展示时机
3. **广告类型搭配**: 组合使用Banner、插屏、激励视频
4. **季节性调整**: 根据用户活跃时间调整策略
