# 全球数据保护法规合规指南

## 欧盟 GDPR (General Data Protection Regulation)

### 📋 核心要求

#### 数据处理原则
- **合法性**: 必须有合法依据（同意、合同、合法利益等）
- **目的限制**: 数据处理目的明确、合法、正当
- **数据最小化**: 只处理必要的个人数据
- **准确性**: 确保数据准确并及时更新
- **存储限制**: 不超过必要期限存储数据
- **完整性和保密性**: 确保数据安全

#### 用户权利
```markdown
## GDPR 用户权利实现检查
- [ ] 知情权: 隐私政策详细说明数据处理
- [ ] 访问权: 提供个人数据副本功能
- [ ] 更正权: 允许用户修正错误信息
- [ ] 删除权: 用户可要求删除数据（被遗忘权）
- [ ] 限制处理权: 用户可限制某些数据处理
- [ ] 数据携带权: 提供数据导出功能
- [ ] 反对权: 用户可反对基于合法利益的处理
```

#### 技术和组织措施
- **数据保护设计**: Privacy by Design
- **默认数据保护**: Privacy by Default  
- **数据保护影响评估**: 高风险处理需要DPIA
- **数据处理记录**: 详细记录处理活动
- **数据泄露通知**: 72小时内报告监管机构

### 💰 违规罚款
- **最高**: 年收入4%或2000万欧元（取较高者）
- **严重违规**: 基本原则违反、用户权利侵犯
- **一般违规**: 年收入2%或1000万欧元

## 中国数据保护法律

### 📱 个人信息保护法 (PIPL)

#### 核心原则
- **合法、正当、必要**: 处理个人信息的基本原则
- **诚信原则**: 不得欺骗、误导个人
- **目的明确**: 处理目的具体、明确、合理
- **最小必要**: 限制在实现目的最小范围

#### 敏感个人信息
```markdown
## 敏感信息处理要求
敏感个人信息包括:
- 生物识别信息 (指纹、面部识别)
- 宗教信仰、健康、性取向
- 14岁以下未成年人个人信息
- 金融账户信息

特殊要求:
- [ ] 具有特定目的和充分必要性
- [ ] 采取严格保护措施
- [ ] 告知处理的必要性及其影响
- [ ] 取得个人单独同意
```

#### 跨境传输
- **数据本地化**: 重要数据原则上在境内存储
- **安全评估**: 向境外提供需要安全评估
- **认证机制**: 通过专门机构认证
- **标准合同**: 签订标准合同条款

### 🔒 网络安全法 (CSL) & 数据安全法 (DSL)
- **关键信息基础设施**: 运营者数据本地化
- **数据分类分级**: 重要数据识别和保护
- **风险评估**: 定期安全风险评估

## 美国隐私法律

### 👶 COPPA (Children's Online Privacy Protection Act)

#### 适用范围
- 面向13岁以下儿童的网站/应用
- 收集13岁以下儿童个人信息的服务

#### 合规要求
```markdown
## COPPA 合规检查
- [ ] 隐私政策清楚说明信息收集实践
- [ ] 获得可验证的家长同意
- [ ] 应家长要求提供孩子信息
- [ ] 应家长要求删除孩子信息
- [ ] 不要求儿童提供超出必要的信息
- [ ] 建立安全程序保护儿童信息
```

#### 家长同意方法
- **书面同意**: 传真、邮寄签名表格
- **信用卡**: 小额信用卡交易
- **数字签名**: 符合电子签名法
- **视频会议**: 视频通话确认
- **政府ID**: 上传身份证件

### 🌴 CCPA (California Consumer Privacy Act)

#### 消费者权利
- **知情权**: 了解收集和使用的个人信息
- **删除权**: 要求删除个人信息
- **选择退出权**: 拒绝出售个人信息
- **非歧视权**: 不因行使权利受到歧视

## 其他重要地区

### 🇧🇷 巴西 LGPD (Lei Geral de Proteção de Dados)
- 类似GDPR的综合隐私法
- 适用于在巴西处理个人数据的组织
- 数据保护官 (DPO) 要求
- 最高罚款：年收入2%

### 🇮🇳 印度 DPDP Act 2023
- **数据主体权利**: 访问、更正、删除、携带
- **跨境转移**: 仅限政府批准的国家
- **儿童数据**: 需要家长同意
- **违规罚款**: 最高250 crore卢比

### 🇸🇬 新加坡 PDPA
- **同意义务**: 收集、使用、披露需同意
- **通知义务**: 告知数据使用目的
- **访问和更正**: 用户可查看和修正数据
- **数据泄露通知**: 重大泄露需通知

### 🇨🇦 加拿大 PIPEDA
- **合理目的**: 数据收集用于合理商业目的
- **知识和同意**: 用户知情并同意
- **限制收集**: 仅收集必要信息
- **准确性**: 确保数据准确完整

## 行业特定法规

### 💊 医疗行业
#### HIPAA (美国)
- **受保护健康信息**: PHI严格保护
- **最小必要原则**: 最少信息访问
- **商业伙伴协议**: BA Agreement必需
- **安全规则**: 电子PHI保护措施

#### MDR (欧盟医疗器械法规)
- **软件医疗器械**: 应用可能属于医疗器械
- **CE标记**: 欧盟市场准入要求
- **临床评价**: 安全性和有效性证明

### 💳 金融行业
#### PCI DSS (支付卡数据安全标准)
- **持卡人数据保护**: 加密存储和传输
- **网络安全**: 防火墙和安全配置
- **定期测试**: 安全系统测试
- **信息安全政策**: 维护安全政策

### 🎓 教育行业
#### FERPA (美国)
- **教育记录保护**: 学生记录隐私
- **家长权利**: 查看和修正记录权利
- **披露限制**: 严格限制记录披露

### 📱 移动应用特殊考虑
- **权限最小化**: 仅请求必要权限
- **透明度**: 清楚说明权限用途
- **用户控制**: 允许权限管理
- **儿童保护**: 年龄验证机制
- **跨境数据**: 明确数据存储位置

### 🔄 持续合规
- **法规更新监控**: 跟踪法规变化
- **定期评估**: 季度或年度合规审查  
- **事件响应**: 数据泄露应急预案
- **员工培训**: 定期隐私安全培训
- **供应商管理**: 第三方合规要求
